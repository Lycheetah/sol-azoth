## PART 2: GRAMMAR SPECIFICATION

### BNF (Backus-Naur Form) Grammar

```bnf
<expression>     ::= <statement>
                   | <expression> <connector> <expression>

<statement>      ::= <transformation>
                   | <query>
                   | <declaration>
                   | <composition>

<transformation> ::= <state> <arrow> <state>
                   | <state> <arrow> <state> <modifier>

<state>          ::= <invariant>
                   | <field>
                   | <composite_state>

<composite_state> ::= <state> <operator> <state>
                    | "(" <expression> ")"

<operator>       ::= <d-class>    // Dynamics: Φ↑, ⊗, →, etc.
                   | <m-class>    // Meta: Z↓, ∫, ∂, etc.

<arrow>          ::= "→"          // Forward transition
                   | "←"          // Reverse transition
                   | "↔"          // Bidirectional

<modifier>       ::= <confidence>
                   | <duration>
                   | <context>

<confidence>     ::= "(" "confidence" "=" number ")"

<duration>       ::= "(" "time" "=" number ")"

<context>        ::= "[" condition "]"

<connector>      ::= ","           // Sequential
                   | ";"           // Ordered sequence
                   | "|"           // Alternative
                   | "∧"           // Parallel
                   | "∨"           // OR (either one)

<query>          ::= "?" <expression>

<declaration>    ::= symbol ":=" <expression>
```

### Type System

**Types:**

```
Type τ ::= Invariant
         | Field
         | Transformation(τ₁ → τ₂)
         | Composition[τ₁, τ₂]
         | Query(τ → Bool)
```

**Type Rules:**

```
(FIELD_OP)
    Ψ : Field    T : Operator
    ─────────────────────────
    Ψ ⊗ T : Field

(TRANSFORM)
    ψ₁ : State    ψ₂ : State

## PART 3: SEMANTIC RULES

All valid LAMAGUE expressions must satisfy these five rules:

### Rule 1: Entropy Conservation

**Statement:** Transformations cannot increase total entropy.

**Formula:**
```
For all T ∈ D-Class:
  S(T(ψ)) ≤ S(ψ) + S_external

Where S_external accounts for heat flow from environment
```

**Implication:** You can't create order from nothing; any local order requires external energy input.

**Example:**
```
Valid: ∅ → Ψ (nothing → consciousness)
  Energy cost: Must provide external energy

Invalid: Ψ_inv → ∅ (consciousness → nothing without energy)
  Would violate entropy conservation
```

---

### Rule 2: Invariant Preservation

**Statement:** Core invariants must survive all transformations.

**Formula:**
```
For all T ∈ D-Class:
  Inv(ψ) ⟹ Inv(T(ψ))

Where Inv are the seven constitutional invariants
```

**Implication:** Transformation can change many properties but must preserve core identity.

**Example:**
```
Human Primacy (Invariant 1) must survive all AI transformations
(AI learns, evolves, changes) → (humans still in control)

If Inv would be violated, transformation is blocked.
```

---

### Rule 3: Coherence Requirement

**Statement:** Composed operations must maintain coherence.

**Formula:**
```
Φ(g ∘ f) ≥ min(Φ(f), Φ(g))

Where Φ = coherence measure
```

**Implication:** Composition doesn't degrade overall system coherence below its parts.

**Example:**
```
If Process A has coherence 0.8
And Process B has coherence 0.9
Then (B ∘ A) must have coherence ≥ 0.8

The weakest link doesn't get weaker.
```

---

### Rule 4: Causality

**Statement:** Temporal transformations must respect causality.

**Formula:**
```
For temporal operators T_t:
  T_t₂ ∘ T_t₁ defined only if t₂ ≥ t₁

Cannot apply future operations before past ones.
```

**Implication:** Time is directed; can't reverse causal order.

---

### Rule 5: Reversibility by Default

**Statement:** Transformations should be reversible unless explicitly marked irreversible.

**Formula:**
```
For T ∈ D-Class:
  ∃ T⁻¹ such that T⁻¹ ∘ T = id (identity)

Unless marked: T↔ (irreversible arrow)
```

**Implication:** Preserve optionality; enable undo.

## PART 5: COMPOSITIONAL SEMANTICS

### Sequential Composition

**Syntax:**
```
T₁ → T₂ → T₃ → ... → Tₙ
```

**Semantics:**
```
Apply T₁ to state ψ₀:
  ψ₁ = T₁(ψ₀)

Apply T₂ to result:
  ψ₂ = T₂(ψ₁)

Apply T₃:
  ψ₃ = T₃(ψ₂)

...continue...

Result: ψ_final = Tₙ(...T₃(T₂(T₁(ψ₀)))...)
```

**Example:**
```
LAMAGUE: Ao → Φ↑ → Ψ_inv
Apply:
  Step 1: Reset to baseline (Ao)
  Step 2: Grow/ascend (Φ↑)
  Step 3: Reach goal (Ψ_inv)
```

### Parallel Composition

**Syntax:**
```
T₁ ∧ T₂
```

**Semantics:**
```
Apply both transformations independently:
  ψ'₁ = T₁(ψ₁)
  ψ'₂ = T₂(ψ₂)

Result: (ψ'₁, ψ'₂)

No ordering, no interaction.
```

**Example:**
```
LAMAGUE: Ψ↑ ∧ S↓
Meaning: "Growth and entropy reduction happening simultaneously"
Reality: Different processes running in parallel
```

### Conditional Application

**Syntax:**
```
if P(Ψ) then T₁ else T₂

Or more concisely:
[P(Ψ)] T₁ | ¬P(Ψ) T₂
```

**Semantics:**
```
Check condition P on current state:
  if P(ψ) = True:
    return T₁(ψ)
  else:
    return T₂(ψ)
```

**Example:**
```
LAMAGUE: [Ψ < threshold] Φ↑ | [Ψ ≥ threshold] Φ↓
Meaning: "If below threshold, ascend; if at/above threshold, descend"
Use: Homeostatic regulation (stay around set point)
```

        'GRADIENT': r'∇',

        # Meta
        'Z_DOWN': r'Z↓',
        'Z_RIGHT': r'Z→',
        'Z_SPIRAL': r'Z↺',
        'INTEGRAL': r'∫',
        'PARTIAL': r'∂',

        # Connectors
        'COMMA': r',',
        'SEMICOLON': r';',
        'PIPE': r'\|',
        'AND': r'∧',
        'OR': r'∨',

        # Special
        'QUERY': r'\?',
        'LPAREN': r'\(',
        'RPAREN': r'\)',
        'LBRACKET': r'\[',
        'RBRACKET': r'\]',
    }

    def tokenize(self, expr: str) -> List[Token]:
        tokens = []
        i = 0
        while i < len(expr):
            # Try each token type
            for token_type, pattern in self.TOKENS.items():
                match = re.match(pattern, expr[i:])
                if match:
                    tokens.append(Token(
                        type=token_type,
                        value=match.group(0),
                        pos=i
                    ))
                    i += match.end()
                    break
            else:
                # No match; skip whitespace or raise error
                if expr[i].isspace():
                    i += 1
                else:
                    raise LexerError(f"Unknown token at {i}: {expr[i]}")

        return tokens
```

### Syntax Analysis (Parsing)

```python
class LAMAGUEParser:
    def parse(self, tokens: List[Token]) -> AST:
        self.tokens = tokens
        self.pos = 0
        return self.parse_expression()

    def parse_expression(self) -> AST:
        left = self.parse_statement()

        while self.pos < len(self.tokens) and self.tokens[self.pos].type in ['COMMA', 'SEMICOLON', 'AND', 'OR']:
            op = self.tokens[self.pos].type
            self.pos += 1
            right = self.parse_statement()
            left = BinaryOp(op, left, right)

        return left

    def parse_statement(self) -> AST:
        if self.current_token().type == 'QUERY':
            self.pos += 1
            return Query(self.parse_expression())

        # Parse transformation: state → state
        source = self.parse_state()

        if self.current_token().type in ['ARROW_FWD', 'ARROW_REV']:
            arrow = self.current_token().type
            self.pos += 1
            target = self.parse_state()
            return Transformation(source, arrow, target)

        return source

    def parse_state(self) -> AST:
        token = self.current_token()

        if token.type in ['PSI_INV', 'ANCHOR', 'VOID', 'OMEGA',
                         'PSI', 'ENTROPY', 'ANCHOR_FIELD', 'COHERENCE', 'VARIANCE', 'GRADIENT']:
            self.pos += 1
            return State(token.value)

        elif token.type == 'LPAREN':
            self.pos += 1
            expr = self.parse_expression()
            if self.current_token().type != 'RPAREN':
                raise ParseError(f"Expected ), got {self.current_token()}")
            self.pos += 1
            return expr

        else:
            raise ParseError(f"Expected state, got {token}")

    def current_token(self) -> Token:
        if self.pos < len(self.tokens):
            return self.tokens[self.pos]
        return Token(type='EOF', value='', pos=-1)
```
